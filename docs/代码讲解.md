# CineMatch 项目代码讲解文档

> 本文档对项目中**每一个文件和关键函数**逐一讲解其作用、设计意图和调用关系。

## `config/`

**配置管理模块 —— 集中管理所有环境变量、路径、模型参数**

### `settings.py`

应用配置中心（Pydantic BaseSettings），支持 .env 文件和环境变量覆盖。
- `Settings` 类: 定义所有配置项（API端口、模型参数、缓存TTL等）
- `get_settings()`: 单例模式获取配置实例（lru_cache）
- `movies_path / credits_path`: 数据文件路径的 property

### `logging_config.py`

日志配置。
- `setup_logging()`: 配置全局日志格式和级别

## `src/data/`

**数据管线模块 —— 加载、清洗、关联、缓存**

### `loader.py`

TMDB 数据加载器。
- `DataLoader.load_movies()`: 读取 tmdb_5000_movies.csv
- `DataLoader.load_credits()`: 读取 tmdb_5000_credits.csv
- `DataLoader.get_merged_data()`: 按 id 合并两个数据集

### `preprocessor.py`

特征预处理器（核心模块，被所有推荐器依赖）。
- `DataPreprocessor.preprocess()`: 完整预处理管线（5步）
- `_parse_json_columns()`: 解析 genres/cast/crew 等 JSON 字符串为 Python list
- `_extract_features()`: 从 list of dict 提取 name（genres_list, cast_list, director）
- `_clean_features()`: 小写化+去空格（'Robert Downey Jr.' → 'robertdowneyjr'）
- `_handle_missing_values()`: 填充缺失值（overview→空串, runtime→0, list→空list）
- `get_movie_by_title()`: O(1) 按标题查电影

### `movielens_loader.py`

MovieLens 1M 数据加载器 + TMDB 关联器。
- `MovieLensLoader.build_link_map()`: 通过标题模糊匹配关联 MovieLens movieId → TMDB index（873部匹配）
- `MovieLensLoader.build_rating_matrix()`: 构建真实的 user×movie 评分矩阵（替代模拟数据）
- `MovieLensLoader.train_test_split()`: 按用户 leave-k-out 拆分训练/测试集

### `cache_manager.py`

相似度矩阵缓存管理器。
- `CacheManager.get/set()`: 内存+磁盘二级缓存（pickle）
- `CacheManager.cleanup_expired()`: 自动清理过期缓存（TTL 机制）

## `src/core/`

**推荐算法核心模块 —— 6种推荐器 + 1个抽象基类**

### `base_recommender.py`

推荐器抽象基类（所有推荐器必须实现）。
- `fit(movies_df)`: 训练模型
- `recommend(title, top_n)`: 根据电影标题推荐 top-N
- `get_similarity_score(title1, title2)`: 获取两部电影的相似度
- `find_similar_titles()`: 模糊搜索电影标题
- `_format_recommendations()`: 格式化推荐结果为 DataFrame

### `content_based.py`

基于内容的推荐（TF-IDF + Cosine Similarity）。
- `fit()`: 对电影简介做 TF-IDF 向量化（5000维, 1-2gram），计算全量余弦相似度矩阵
- `recommend()`: 从相似度矩阵取 top-N
- `get_top_keywords()`: 获取电影的 TF-IDF 关键词
- `get_common_keywords()`: 获取两部电影的共同关键词（几何均值）

### `metadata_based.py`

基于元数据的推荐（加权特征匹配）。
- `_create_soup()`: 将 genres×3 + director×3 + cast×2 + keywords×1 拼接为"soup"文本
- `fit()`: 用 CountVectorizer 编码 soup，计算余弦相似度
- `get_matching_features()`: 返回两部电影的共同导演/类型/演员/关键词
- `get_feature_contributions()`: 计算各特征对相似度的贡献比例

### `collaborative.py`

协同过滤推荐（3种方法）。
- `_try_load_movielens()`: 优先加载 MovieLens 真实评分，找不到则回退模拟数据
- `_build_item_based()`: Item-CF（电影间余弦相似度 + KNN）
- `_build_user_based()`: User-CF（用户间余弦相似度 + 邻居聚合）
- `_build_svd()`: 截断 SVD 矩阵分解（50个隐因子）
- `recommend_for_user()`: 为特定用户生成个性化推荐

### `hybrid.py`

混合推荐器（核心入口）。
- `fit()`: 同时训练 Content + Metadata + CF 三个子推荐器
- `_compute_hybrid_similarity()`: 按权重融合三个相似度矩阵
- `recommend()`: 返回混合推荐 + 可选的逐方法得分分解
- `set_weights()`: 动态调整方法权重（前端滑块联动）
- `compare_methods()`: 4种方法并排对比同一查询

### `demographic.py`

人口统计学推荐（IMDB加权评分，冷启动兜底）。
- IMDB公式: score = v/(v+m)·r + m/(v+m)·c
- `recommend()`: 返回全局评分最高的 top-N（非个性化）
- `get_top_movies()`: 支持按类型/最低票数筛选

### `knn_svd_ensemble.py`

KNN+SVD 集成推荐。
- `fit()`: 建立 User-KNN + SVD 预测矩阵
- `recommend_for_user()`: 1)KNN找相似用户 2)收集候选电影 3)SVD预测评分重排序
- `recommend()`: 按标题的 fallback（使用 SVD item-factor 相似度）

## `src/prediction/`

**票房预测模块 —— LightGBM + XGBoost + CatBoost 三模型集成**

### `base_predictor.py`

预测器抽象接口。
- `fit()`: 训练
- `predict()`: 预测票房
- `save() / load()`: 模型持久化

### `box_office_predictor.py`

票房预测器（Kaggle Top 1-3% 方案复刻）。
- `load_data()`: 加载 Kaggle 竞赛数据 + 外部特征 + 手动数据修正
- `_build_features()`: 55维特征工程（budget/popularity/genres/release_date/target_encoding等）
- `fit()`: 5-fold CV 训练 LGB+XGB+Cat，记录每折 train/val loss 曲线
- `predict()`: 加权集成预测（Cat 55% + LGB 25% + XGB 20%）
- `feature_importance()`: LightGBM gain 特征重要度
- `save() / load()`: pickle 序列化模型到 data/cache/
- RMSLE: 1.79, Train-Val Gap: ~0.5（健康水平）

## `src/evaluation/`

**评估模块 —— 基于 MovieLens 真实评分的离线评估**

### `evaluator.py`

推荐质量评估器。
- `__init__()`: 加载评分矩阵 + 按用户 80/20 拆分 train/test
- `evaluate()`: 对单个推荐器计算 7 种指标（Precision/Recall/NDCG/MAP/Coverage/Novelty/Diversity）
- `evaluate_all_methods()`: 批量评估 Content/Metadata/CF/Hybrid 4种方法

## `src/explainability/`

**可解释性模块 —— 让推荐结果可理解**

### `rule_based.py`

基于规则的解释器。
- `explain_metadata_based()`: 检查同导演/同类型/共同演员/关键词，生成自然语言解释
- `explain_content_based()`: 找共同 TF-IDF 关键词
- `explain_collaborative_filtering()`: 统计共同粉丝比例
- `explain_hybrid()`: 分析各方法的贡献占比

### `shap_explainer.py`

SHAP 特征归因解释器。
- `prepare_features()`: 构建电影的 one-hot 特征矩阵
- `train()`: 训练 GBM 代理模型拟合相似度矩阵
- `explain_pair()`: 用 TreeExplainer 计算两部电影间的 SHAP 值
- `get_feature_importance()`: 全局特征重要度

### `visualization.py`

解释可视化工具。
- `plot_shap_waterfall()`: SHAP 瀑布图
- `plot_shap_force()`: 力图（推/拉贡献）
- `plot_feature_importance()`: 特征重要度柱状图
- `plot_method_comparison()`: 方法对比柱状图

## `src/services/`

**业务逻辑层 —— API 和前端的中间层**

### `recommendation_service.py`

推荐服务（主入口）。
- `initialize()`: 加载数据 + 初始化所有推荐器 + 解释器
- `get_recommendations()`: 统一入口，支持 content/metadata/cf/hybrid 方法切换
- `explain_recommendation()`: 调用解释器生成推荐原因

### `search_service.py`

电影搜索服务。
- `search()`: 标题搜索 + 多维过滤（类型/年份/评分/导演）+ 分页
- `fuzzy_search()`: 模糊匹配（SequenceMatcher）
- `get_suggestions()`: 自动补全

### `user_service.py`

用户管理服务。
- CRUD 操作 + 评分 + 收藏夹 + 偏好设置
- JSON 文件存储（轻量级，无需数据库）

### `analytics_service.py`

数据分析服务。
- `get_overview()`: 全局统计
- `get_genre_statistics()`: 类型分布
- `get_budget_revenue_analysis()`: 预算/票房分析

## `src/utils/`

**工具库 —— 评估指标、相似度计算、文本处理**

### `metrics.py`

7 种评估指标实现：
- `calculate_precision/recall/f1()`: 基础精度指标
- `calculate_ndcg()`: 归一化折损累积增益（考虑排序质量）
- `calculate_map()`: 平均精度均值
- `calculate_coverage()`: 目录覆盖率
- `calculate_novelty()`: 新颖度（逆热门度）
- `calculate_diversity()`: 列表内多样性

### `similarity.py`

相似度计算函数集合

### `text_processing.py`

文本预处理工具（tokenize, clean, stem）

## `api/routes/`

**API 路由处理器**

推荐端点。

### `movies.py`

电影端点。
- `GET /`: 搜索+筛选+分页
- `GET /popular`, `GET /top-rated`
- `GET /suggestions`: 自动补全
- `GET /by-genre/{genre}`, `GET /by-director/{director}`
- `GET /analytics/overview`: 数据统计

### `users.py`

用户端点。
- CRUD + 评分 + 收藏夹 + 偏好 + 个性化推荐

### `health.py`

健康检查。
- `GET /health`: 服务状态 + 模型加载状态
