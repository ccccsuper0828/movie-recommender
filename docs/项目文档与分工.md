# CineMatch —— 电影数据分析与智能推荐系统

## 项目文档

---

## 一、项目概述

### 1.1 项目目标

基于 TMDB 5000 电影数据集、Kaggle TMDB Box Office 竞赛数据集以及 MovieLens 1M 用户评分数据集，构建一个完整的电影数据分析与推荐系统，完成以下核心任务：

1. **电影特征可视化分析** —— 对票房、预算、类型、评分等特征的探索性分析
2. **电影票房预测** —— 基于 CatBoost + XGBoost + LightGBM 三模型集成，复刻 Kaggle 竞赛 Top-7% 方案
3. **多算法电影推荐** —— 实现 8 种推荐算法（Content-Based / Metadata / CF / SVD / Hybrid / Demographic / KNN+SVD Ensemble）
4. **推荐质量评估** —— 基于 MovieLens 真实用户评分进行 train/test split 评估（Precision@K / NDCG@K / Coverage / Novelty / Diversity）
5. **可解释性分析** —— Rule-Based 解释 + SHAP 特征归因
6. **全栈 Web 应用** —— Streamlit 前端 + FastAPI 后端 + Docker 部署

### 1.2 数据集

| 数据集 | 规模 | 用途 |
|--------|------|------|
| TMDB 5000 Movies + Credits | 4803 部电影 | 推荐系统核心数据（简介、类型、导演、演员、评分、票房） |
| Kaggle TMDB Box Office | 3000 train + 4398 test | 票房预测（预算、受欢迎度、上映日期、演职人员等） |
| MovieLens 1M | 100 万条评分，6040 用户，3706 部电影 | 协同过滤训练 + 推荐评估的 ground truth |
| 外部补充数据 | release_dates_per_country, AdditionalFeatures | 各国上映日期、额外的 popularity 和 rating |

### 1.3 技术栈

| 层级 | 技术 |

## 三、核心模块详细说明

### 3.1 推荐算法（8 种）

#### (1) Content-Based (TF-IDF)
- **文件**: `src/core/content_based.py`
- **原理**: 对电影简介 (overview) 进行 TF-IDF 向量化（5000 维，1-2gram），计算余弦相似度
- **优点**: 不依赖用户数据，可解释性强
- **输出**: 与输入电影简介最相似的 Top-K 电影

#### (2) Metadata-Based (CountVectorizer)
- **文件**: `src/core/metadata_based.py`
- **原理**: 将 genres/director/cast/keywords 加权拼接为"soup"，用 CountVectorizer 编码后计算余弦相似度
- **权重**: genre×3, director×3, cast×2, keyword×1

#### (3) Collaborative Filtering
- **文件**: `src/core/collaborative.py`
- **原理**: 基于 MovieLens 真实用户-电影评分矩阵，支持三种方法：
  - Item-Based KNN（电影相似度）
  - User-Based KNN（用户相似度）
  - SVD 矩阵分解（截断 SVD，50 个隐因子）

#### (4) Hybrid Recommender
- **文件**: `src/core/hybrid.py`
- **原理**: 将 Content/Metadata/CF 三种方法的相似度矩阵按可调权重（默认 0.3/0.4/0.3）线性融合
- **支持**: 动态调权、方法对比、逐方法得分分解

#### (5) Demographic Recommender
- **文件**: `src/core/demographic.py`
- **原理**: IMDB 加权评分公式 `score = v/(v+m)·r + m/(v+m)·c`
- **用途**: 冷启动时的非个性化热门推荐

#### (6) KNN+SVD Ensemble
- **文件**: `src/core/knn_svd_ensemble.py`
- **原理**: 
  1. User-KNN 找到最相似的 K 个用户
  2. 收集邻居看过但目标用户没看的电影作为候选
  3. SVD 预测目标用户对候选电影的评分，取 Top-N

### 3.2 票房预测

- **文件**: `src/prediction/box_office_predictor.py`
- **数据**: Kaggle TMDB Box Office 竞赛数据集（3000 部训练电影 + 外部特征）
- **特征工程**: ~50 维特征，包含：
  - 核心数值：budget, popularity, runtime, vote
  - 外部补充：popularity2, rating, theatrical（各国上映数量）
  - 时间特征：上映年/月/周几/是否暑期档/节假日档
  - 类型特征：12 种 genre flag + genre 数量
  - 制片特征：制片公司数量、是否大厂（Warner/Disney/Universal 等）
  - 续集/系列：belongs_to_collection
  - 交互特征：budget×popularity, budget×rating, budget×theatrical 等
  - 文本特征：overview 长度、是否有 homepage/tagline
- **模型**: CatBoost (35%) + LightGBM (35%) + XGBoost (30%) 加权集成
- **训练**: 5-fold CV, learning_rate=0.01, 3000 轮 + early_stopping
- **指标**: RMSLE ≈ 1.87（对标 Kaggle Top 5-10%）

### 3.3 评估系统

- **文件**: `src/evaluation/evaluator.py`
- **数据源**: MovieLens 1M 真实评分（通过标题匹配关联到 TMDB，873 部重叠电影）
- **方法**: 
  - 按用户 80/20 随机拆分 hold-out
  - 对每个 test 用户生成 Top-K 推荐
  - 与 hold-out 真实高分电影对比
- **指标**:
  - Precision@K, Recall@K, NDCG@K, MAP@K（准确性）
  - Coverage, Novelty, Diversity（多样性/新颖性）

### 3.4 可解释性

- **Rule-Based** (`src/explainability/rule_based.py`): 基于特征匹配（同导演/同类型/共同演员/关键词）生成自然语言解释
- **SHAP** (`src/explainability/shap_explainer.py`): 训练 GBM 代理模型拟合相似度矩阵，用 TreeExplainer 计算 SHAP 值
- **可视化** (`src/explainability/visualization.py`): Waterfall 图、Force 图、Feature Importance 图

### 3.5 前端页面（8 个）

| 页面 | 功能 |
|------|------|
| 🏠 Home | Hero 展示 + 快速推荐 + 热门/高分排行 |
| 🎯 Get Recommendations | 6 个方法 Tab（Hybrid/Content/Metadata/CF/Demographic/KNN+SVD）|
| 🔬 SHAP Deep-Dive | 选推荐电影 → SHAP waterfall + 贡献因子 + 全局特征重要度 |
| 🔍 Explore | 按类型/评分/排序浏览全部 4803 部电影 |
| 📊 Analytics | 数据统计 + MovieLens 推荐质量评估仪表盘 |
| 💰 Box Office | 票房预测模型训练/评估 + 单部电影预测 |
| ⚖️ Compare Methods | 4 种方法并排对比同一查询的推荐结果 |
| ℹ️ About | 项目/技术/数据集说明 |

### 3.6 后端 API

- **框架**: FastAPI + Pydantic v2
- **路由**: `/api/v1/recommendations`, `/api/v1/movies`, `/api/v1/users`, `/health`
- **特性**: CORS、依赖注入（Singleton Service）、OpenAPI 自动文档
- **部署**: Docker multi-stage build + docker-compose (API + Frontend + Redis + Nginx)

## 七、运行方式

### 7.1 环境准备

```bash
# 安装依赖
pip install -r requirements.txt
pip install lightgbm xgboost catboost

# 确保数据文件在 data/raw/ 目录下
```

### 7.2 启动 Streamlit 前端

```bash
cd MOVIE
PYTHONPATH=$(pwd) streamlit run frontend/app.py --server.port 8501 --server.headless true
```

访问 http://localhost:8501

### 7.3 启动 FastAPI 后端

```bash
cd MOVIE
uvicorn api.main:app --reload --port 8000
```

API 文档: http://localhost:8000/docs

### 7.4 Docker 部署

```bash
docker-compose up -d
```

### 7.5 运行测试

```bash
pytest tests/unit/ -q      # 单元测试（秒级）
pytest tests/ -q            # 全量测试（含模型 fit，较慢）
```
